Menu

Studio

2014


Background

'Jocoos' ran a web service called Studio, which provides auditions for the agency. At the time, I found the web service far from user-centered design. So I did a personal project to redesign it considering the user's position.


Anaylsis

I defined who the users were using this service and analyzed each user's needs and tasks.

Service Map

Based on this, I organized the functions and menus required for studio services and designed the overall service structure.

Design
Audition List

AS-IS

  1. It is difficult for users to recognize GNB's main menu and account-related menu at once.
  2. GNB's audition creation button has little correlation with each page.
  3. There are often no audition images.
  4. There is an inconvenience of going into the detail to check the key information of the audition.
  5. There is an inconvenience of not being able to see the classification according to the audition status in chronological order.

TO-BE

  1. Position the GNB to the left and insert an icon to make it easier for users to recognize it even if it changes variable.
  2. Login information is added to make it easier to distinguish between main menu and account related menu.
  3. It is easy to distinguish between auditions by focusing on the title of the audition and by adding a category icon.
  4. Analyze the importance of audition information so that key audition information is exposed even if it does not enter the world.
  5. It provides a filter function that can be classified according to the audition status for ease of viewing.


Audition Detail

AS-IS

  1. The GNB and the left detailed menu are confusing to users.
  2. The detailed menu is located under the audition thumbnail, making it difficult to move between menus.
  3. The same information as the audition is unnecessarily repeated.
  4. Information is inconvenient because of the mixed layout configuration.

TO-BE

  1. Position the GNB to the left and insert an icon to make it easier for users to recognize it even if it changes variable.
  2. The detailed menu is placed under the audition title to make it easier to move between menus.
  3. Eliminate the audition list information and repeated information and unify the information format.
  4. A single-page grid makes it easier for users to see at a glance.


Video List

AS-IS

  1. The upper GNB and image category classification areas cover many areas.
  2. In order for the user to scroll down and sort images, they must go back to the top.
  3. Besides the video thumbnail, the title and the author, the necessary image information cannot be viewed at a glance.
  4. Watch and contact functions are less frequently used in the image list.

TO-BE

  1. The categories and classification menus are located to the right, allowing users to access the menus at once even if they scroll.
  2. Categorize the image information so that users can see the information they need at a glance.
  3. Re-defined the Watch and Contact functions so that images can be captured and viewed later by adding a collection function that can occur from the image list.


Video Detail

AS-IS

  1. It is inconvenient to view the contents of the right detailed menu tab in pop-up format.
  2. Video and Comments don't seem to have to be divided on the detailed menu tab.
  3. Although images and profiles of applicants are the most important information, it is hard to see them at a glance.
  4. It is not possible to determine who did it if other hosts/managers were watch, contact.

TO-BE

  1. Vertical screen segmentation allows scrolling in each area.
  2. Divide the applicant image information into the profile area and the image information area to facilitate the separation of information.
  3. For the applicant profile, add a button to view it as a new window.
  4. Displays host/manager information when Watch and Contact functions are enabled.
